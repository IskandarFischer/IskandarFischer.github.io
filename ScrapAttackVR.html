Scrap Attack VR

Scrap Attack VR On Steam

During the project I had several different roles including various gameplay features, porting the game to different platforms, handling the bridge between the server and the clients and providing help when needed for the database development and finally being part of the Live DevOps team in the final development of Scrap Attack.

A few of the biggest challenges that I faced during the development of Scrap Attack were redesigning several features and scripts to be more efficient and easier to use for both programmers and designers. I was the member of the team which had the most impact on the games performance by working closely with artist and finding ways of modifying the art assets to increase performance, and looking at the profiler to find and redesign scripts which took too much time to execute. Designing new features especially one pertaining to recording in-game footage for the making of the Scrap Attack VR trailers. Porting Botanika to the standalone game Scrap Attack VR and the porting to UWP and later adding compatibility to Vive, Oculus, Windows Mixed Reality. Worked on solving challenges with publishing the game to the Steam Store and Microsoft Store. Another challenge that arose was the creation of a trailer for the game which required a redesign of the designed filming architecture. One of the biggest challenges however was working on a big project that involved many dependencies and a large code base.

While facing the challenges of feature creation, as a team, we decided how the architecture of the solution would be and how the delegation of the task would be done. Once the initial feature design meeting would be completed, I would start creating an architecture and solutions to the delegated feature. Once I designed a robust architecture, I implemented it and relayed the feature with the design team to ensure that the delivered feature is enjoyable and fitting to the requirements, I also relayed the feature with the artist to allow for an easier incorporation of the assets by modifying the code or discussing with the artist as to what is the best way to design the incorporation of the assets. This allowed for me to inspect the profiler and diagnosed scripts or assets which were inefficient and report the problems by adding them to Redmine and solving the issues by working with artist or redesigning the scripts to be more efficient.

A lot of challenges arose when dealing with scripts ported over from Botanika as they had very poor architecture. By redesigning the big features and providing a new solution to the problem, the team were able to create features using the designed features much more easily and iterate on features far more quickly than before speeding up development time significantly. While porting the game several challenges arose.

Firstly the old networking architecture didn’t allow compilation for UWP, this was solved by removing a lot of the networking code and by allowing scripts to bypass the networking features at the lower level. Another issue that arose was the previous architecture didn’t allow for Vive, Oculus or Windows Mixed Reality headsets to be used. I redesigned and implemented a new input handler to allow for any type of inputs to be handled. In addition, several challenges arose when adding different headsets as there were a lot of conflicting scripts, and combining several of the different scripts for each headsets into one. In addition, a lot of different statements depending on which headset or compilation method we’re created to smartly choose which Key-Bindings to use.

While publishing the game to steam, I worked on integrating several steam features to the game, solved issues with how scripts didn’t allow to be compiled due to having to publish to both the Windows Store and Steam as some scripts wouldn’t pass Microsoft Certification, and finally learnt the build pipeline for both the Windows Store and Steam and actively published the builds to the stores.

During the end of the project I directed, filmed, and edited the newest Scrap Attack Trailer while also designing several levels by myself and leading the art direction of those levels.

The project was presented to Google.
